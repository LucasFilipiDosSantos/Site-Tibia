<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → cart/checkout API | Untrusted request payloads and JWT claims drive cart/order creation flows. |
| API exception mapping → public ProblemDetails | Internal conflict details cross boundary into external response bodies. |
| API order read endpoint → persisted snapshot model | Risk of exposing mutable live catalog state in historical reads. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-12 | E | Checkout/cart endpoints | mitigate | Require authenticated principal and derive customer identity from JWT `sub` for all cart/checkout operations. |
| T-04-13 | I | Conflict payload rendering | mitigate | Emit only per-line product/requested/available quantities in `lineConflicts`; avoid leaking infrastructure internals. |
| T-04-14 | T | Checkout atomicity at API boundary | mitigate | Preserve service-level fail-all semantics and assert no partial order/cart changes on conflict (D-14). |
| T-04-15 | R | Historical order read accuracy | mitigate | Serve snapshot-backed fields from persisted order items only (D-12). |
</threat_model>
