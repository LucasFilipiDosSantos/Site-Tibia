<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| checkout client payload → checkout service | Untrusted delivery instruction and cart/identity input drives order creation. |
| checkout service → inventory reservation | External reserve failures can create partial-state risk if not atomic. |
| order persistence → historical read model | Mutable source joins can corrupt historical price/product truth. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | T | Checkout order snapshot | mitigate | Freeze `UnitPrice`, `Currency`, `ProductName`, `ProductSlug`, `CategorySlug` into immutable snapshot entity (D-09..D-11). |
| T-04-05 | D | Checkout reserve orchestration | mitigate | Fail whole checkout on any line reserve conflict and avoid partial persistence (D-14). |
| T-04-06 | I | Delivery instruction contract | mitigate | Validate required fields per fulfillment type and reject incomplete payloads (D-05..D-08). |
| T-04-07 | R | Historical order reads | mitigate | Serve historic values from stored snapshots only, never live catalog joins (D-12). |
</threat_model>
