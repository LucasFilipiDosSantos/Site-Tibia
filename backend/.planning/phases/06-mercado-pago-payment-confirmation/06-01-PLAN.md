<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Authenticated customer request -> payment-init endpoint | User-controlled order ID enters payment creation path. |
| Internal payment service -> Mercado Pago SDK | Outbound provider request carries monetary/order integrity data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | T | Payment preference external reference binding | mitigate | Force `external_reference=orderId` only; no client-supplied external reference field. |
| T-06-02 | I | Cross-customer order access on payment-init | mitigate | Resolve customer from JWT `sub`, enforce order ownership before creating preference. |
| T-06-03 | R | Payment reconciliation auditability | mitigate | Persist payment-link snapshot (`orderId`, `preferenceId`, expected amount/currency, timestamp). |
| T-06-04 | D | Provider call failure ambiguity | mitigate | Return deterministic failure contract via exception mapping; never create partial paid state on preference failure. |
</threat_model>
