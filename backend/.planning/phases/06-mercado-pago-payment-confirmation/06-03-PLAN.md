## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Verified async payment event -> order lifecycle service | External payment status influences privileged order state transition. |
| Non-approved/invalid payment outcomes -> audit state | Rejected/untrusted events must remain non-mutating but observable. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-10 | E | Paid transition authority | mitigate | Call lifecycle `ApplySystemTransitionAsync` only for verified approved/processed statuses (D-09). |
| T-06-11 | T | Non-approved status coercion to Paid | mitigate | Explicit status mapping keeps pending/in_process/authorized/rejected/cancelled/refunded as non-paid (D-10, D-11). |
| T-06-12 | D | Duplicate paid confirmations causing timeline inflation | mitigate | If already paid, treat as no-op lifecycle branch and persist audit-only outcome (D-12). |
| T-06-13 | R | Missing evidence for skipped transitions | mitigate | Persist processing outcome and reason for every webhook branch, including no-op/rejected paths. |
</threat_model>
