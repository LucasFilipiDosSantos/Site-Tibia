| client→API | Untrusted registration payload can trigger validation/business exceptions crossing HTTP boundary. |
| app-exception→HTTP-response | Internal exception objects are transformed into client-visible error payloads. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06-01 | I | Registration error response | mitigate | Return RFC7807 ProblemDetails without stack traces/raw exception internals for validation and server errors. |
| T-01-06-02 | D | Auth endpoint pipeline | mitigate | Handle expected ArgumentException paths deterministically as 4xx to prevent repeated unhandled exception pipeline failures under weak-password attempts. |
| T-01-06-03 | R | Security-relevant failure diagnosis | accept | Structured ProblemDetails status/type/title is sufficient in this phase; richer audit correlation remains covered by existing SecurityAuditService and later ops phases. |
</threat_model>
