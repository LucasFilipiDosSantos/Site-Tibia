## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Application ports → Infrastructure adapters | Sensitive credentials/tokens cross from business logic to persistence/crypto implementations |
| Infrastructure token issuance → API authorization | Claims and expiry values become security decisions at boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | T | persistence token records | mitigate | Hash refresh/reset/verification tokens before storage and enforce one-time consumed markers |
| T-01-05 | I | JWT claims generation | mitigate | Restrict claim set to required fields and set explicit expiry/issuer/audience validation config |
| T-01-06 | D | session repository paths | mitigate | Add unique indexes + efficient lookups on token hash/normalized email to reduce auth hotspot contention |
</threat_model>
