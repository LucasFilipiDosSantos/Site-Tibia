|----------|-------------|
| API→delivery-provider | Raw verify/reset tokens leave API process and cross into external transport channel. |
| configuration→runtime | Secret SMTP credentials/config enter runtime through config providers/env vars. |
| test-host→delivery-fake | Integration tests override provider to capture transport payloads for verification proof. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07-01 | I | SmtpIdentityTokenDelivery logging | mitigate | Prevent token disclosure by banning raw-token logging and limiting telemetry to metadata (recipient hash/id, expiry timestamp). |
| T-01-07-02 | T | Delivery provider selection/config | mitigate | Enforce strict options validation and startup fail-fast when SMTP provider settings are incomplete or invalid. |
| T-01-07-03 | S | Verification/reset confirm flow | mitigate | Preserve existing token-hash verification and one-time consumption checks in IdentityService confirm paths; tests assert replay rejection. |
| T-01-07-04 | D | SMTP provider transient failures | accept | Phase 01 accepts immediate failure surface; retry/backoff orchestration is handled in later reliability phase (REL-01). |
</threat_model>
