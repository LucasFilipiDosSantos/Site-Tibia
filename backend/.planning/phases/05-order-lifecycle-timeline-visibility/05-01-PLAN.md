## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API/Application transition command → Domain Order aggregate | Untrusted actor intent crosses into lifecycle invariants. |
| Domain timeline event metadata → persistence/API output | Event source/status data becomes auditable external truth. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | E | Order transition authority | mitigate | Enforce source-specific transition matrix in domain/application (`System` owns Paid; customer/admin cancel Pending only). |
| T-05-02 | T | Order status history | mitigate | Append-only event model; no mutation/delete APIs for existing history events. |
| T-05-03 | R | Transition auditability | mitigate | Capture sourceType + from/to states + UTC occurrence on every real transition. |
| T-05-04 | D | Retry/idempotency handling | mitigate | Treat duplicate target-state requests as no-op without extra event writes. |
</threat_model>
